## 报告漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请**不要**通过公开的 Issue 报告。

### 报告流程

1. **私密报告**：请通过 GitHub Security Advisories 报告
   - 访问：https://github.com/GeezCo/gzDoc/security/advisories/new
   - 或发送邮件至：security@geezco.com

2. **提供信息**：
   - 漏洞的详细描述
   - 复现步骤
   - 潜在影响
   - 建议的修复方案（如果有）

3. **响应时间**：
   - 我们会在 **48 小时内**确认收到报告
   - 在 **7 天内**提供初步评估
   - 在 **30 天内**发布修复（视严重程度而定）

### 漏洞等级

我们使用 CVSS 3.1 标准评估漏洞严重程度：

- **严重 (9.0-10.0)**: 立即修复，24小时内发布补丁
- **高危 (7.0-8.9)**: 7天内修复
- **中危 (4.0-6.9)**: 30天内修复
- **低危 (0.1-3.9)**: 下一个版本修复

## 安全最佳实践

### 部署建议

1. **环境变量**
   - 永远不要在代码中硬编码密钥
   - 使用环境变量或密钥管理服务
   - 定期轮换密钥

2. **网络安全**
   - 使用 HTTPS/TLS 1.3
   - 配置防火墙规则
   - 启用 CORS 白名单

3. **数据库安全**
   - 使用强密码
   - 限制数据库访问权限
   - 启用数据库审计日志
   - 定期备份

4. **容器安全**
   - 使用官方基础镜像
   - 定期更新依赖
   - 扫描镜像漏洞
   - 以非 root 用户运行

5. **认证授权**
   - 使用强密码策略
   - 启用多因素认证（MFA）
   - 实施最小权限原则
   - 定期审查权限

### 依赖管理

我们使用以下工具监控依赖漏洞：

- **Java**: Dependabot, OWASP Dependency-Check
- **Python**: Safety, pip-audit
- **Node.js**: npm audit, Snyk

### 安全审计

- 定期进行代码安全审查
- 使用静态代码分析工具（SonarQube）
- 进行渗透测试
- 监控安全公告
