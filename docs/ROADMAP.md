# 📅 GzDoc 开发路线图

> 9个月开发计划 - 从MVP到商业化

---

## 🎯 三大里程碑

### M1: MVP版本 (Month 1-3)
**目标**: 核心功能可演示  
**交付**: 用户系统 + 文档管理 + OCR + 基础RAG  
**标准**: 5秒响应 / 10并发 / Docker部署

### M2: 生产版本 (Month 4-6)
**目标**: 性能达标 + 多租户  
**交付**: 微服务 + Kafka + 计费 + 监控  
**标准**: 3秒P99 / 100并发 / 99%可用性 / K8s部署

### M3: 商业版本 (Month 7-9)
**目标**: 可售卖 + 可私有化  
**交付**: 企业特性 + 完整文档 + 3个客户案例  
**标准**: 1天完成私有化部署 / 90%客户满意度

## 🔄 风险管理

### 技术风险
- **风险1：技术难度超预期** → 降低复杂度，先做MVP
- **风险2：性能达不到要求** → 提前做性能测试，及时优化
- **风险3：AI效果不好** → 多测试Prompt，必要时Fine-tune

### 时间风险
- **风险1：时间不够** → 砍掉非核心功能，聚焦MVP
- **风险2：学习曲线陡峭** → 边做边学，不要提前学
- **风险3：拖延症** → 设定每周目标，公开承诺

### 商业风险
- **风险1：找不到客户** → 提前在社区建立影响力
- **风险2：定价太低** → 前3个客户可以低价，后面涨价
- **风险3：客户需求变化** → 做标准化产品，不做定制
